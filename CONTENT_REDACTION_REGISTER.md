# SmartClover Content Redaction Register

Prepared: 2026-05-11

This register prevents sensitive, previously removed, or credibility-theater language from reappearing in public marketing copy.

| Content | Rule | Allowed Context |
| --- | --- | --- |
| `Andreea Itu` earlier publication name | Remove from homepage and promotional copy. Use only where directly necessary to explain citation continuity. | Neutral citation/reference note, BibTeX/citation material, or evidence register. |
| `Dr. Florian Nicula` | Remove from homepage credibility copy. Use only where directly citing the publication or research context. | Neutral reference section, blog citation context, BibTeX/citation note. |
| `named accountability`, `named leadership`, `public record` | Ban from public marketing copy. | None. |
| `support evaluation by stakeholders` | Ban from public marketing copy. | None. |
| Founder history as persuasion copy | Do not use personal/publication continuity to sell the company. | About-page context only when concise and reader-relevant. |
| TealGuard or named partnership claims | Omit unless dated, approved evidence and permission exist. | Evidence-gated proof page only. |
| Final MDR/approval/certification language | Omit unless final signed evidence exists. | Regulatory page with source, date, and scope. |
| `certified`, `authorized`, or partner-product cybersecurity claims | Use only when wording is mapped to exact personnel scope, partner-product evidence, and approved wording. | Cybersecurity/service page with evidence note, contract discussion, or proof artifact. |
| Clinical outcomes or KPI claims | Omit unless denominators, methods, reporting windows, and approval exist. | Proof page with source and date. |
| Status-management copy | Do not describe SmartClover as if writing an analyst memo about the company. | None in public copy. Replace with reader-value wording. |
